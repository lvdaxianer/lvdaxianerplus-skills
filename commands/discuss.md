# Claude 命令: 需求讨论

手动触发的需求讨论命令。通过交互式提问收集需求的关键信息，并保存到 `.ai/requirements-[功能].md` 文件。

## 使用方法

### 基本用法

```bash
/discuss                    # 启动交互式需求讨论
/discuss 用户登录接口      # 快速模式，直接指定需求概述
```

### 参数

| 参数 | 简写 | 描述 | 必填 |
|------|------|------|------|
| `<需求概述>` | - | 需求的核心内容或标题 | 否 |

## 实现逻辑

### 1. 解析用户输入

- 如果提供了需求概述，直接作为需求详情
- 如果未提供，进入交互模式

### 2. 交互式提问

按顺序引导用户补充：
1. 角色（必填，从 10 个热门角色选择）
2. 使用场景（必填）
3. 角色相关考虑（根据角色不同，10 个热门场景问题）
4. 注意事项（可选）
5. 优先级（必填）
6. 验收标准（可选）
7. 截止日期（可选）
8. 预算/成本（可选）
9. 第三方依赖（可选）
10. 其他约束（可选，如多语言、合规等）

### 3. 呈现确认

将收集的信息整理成标准格式，呈现给用户确认。

### 4. 保存文件

1. 获取当前工作目录
2. 确保 `.ai` 目录存在
3. 读取或创建 `requirements.md`
4. 追加到当天日期分组下
5. 写入文件并告知用户

## 错误处理

- 如果无法创建目录，提示用户检查权限
- 如果无法写入文件，提示用户检查文件权限
- 如果用户取消，告知用户未写入任何内容
- 如果用户长时间无响应，可以重新开始
