# ROUTER-003: Regex 正则路径匹配

## 故事目标

- **核心问题**：验证正则匹配能正确提取路径参数
- **目标用户**：开发者/测试人员
- **预期产出**：正则匹配成功时返回 matched: true 和提取的 params

## 关键节点/里程碑

- **起点**：matchPath 方法接收到 matchType = 'regex'
- **中间步骤**：
  1. 获取 route.path RegExp
  2. 执行 regex.exec(pathname)
  3. 提取命名捕获组
  4. 提取位置捕获组
  5. 组装 params 对象
- **终点**：返回 { matched: true, params: { '1': value, 'groups': ... } }
