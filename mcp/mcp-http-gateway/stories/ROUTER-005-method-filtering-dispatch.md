# ROUTER-005: 方法过滤与路由分发

## 故事目标

- **核心问题**：验证路由能正确过滤 HTTP 方法并分发到对应 handler
- **目标用户**：开发者/测试人员
- **预期产出**：只有匹配方法的请求被处理，其他返回 405

## 关键节点/里程碑

- **起点**：matchAndHandle 方法接收到 method 参数
- **中间步骤**：
  1. 遍历 routes 数组
  2. 检查 route.methods 是否包含 method
  3. 方法匹配时执行路径匹配
  4. 路径匹配时执行 handler
  5. handler 返回 true 则停止
- **终点**：返回 handled: true/false

| 角色 | 职责 | 输入 | 输出 |
|------|------|------|------|
| matchAndHandle | 路由分发 | pathname, method | handled boolean |
| Handler | 处理请求 | req, res | boolean |

## 流程/路径

### 流转条件
route.methods 包含请求 method

### 分支路径
- **条件A** → methods 包含 method → 继续路径匹配
- **条件B** → methods 不包含 method → 跳过该路由
- **条件C** → methods 未定义 → 允许所有方法

### 循环/回退机制
handler 返回 false 时继续匹配下一个路由
