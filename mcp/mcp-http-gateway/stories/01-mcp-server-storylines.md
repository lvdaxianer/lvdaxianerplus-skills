# MCP Server 模块故事线

## 故事线概览

本模块包含 15 条故事线，覆盖 MCP Server 的核心功能。

---

## 故事线 1：STDIO 传输模式启动流程

### 故事目标
- **核心问题**：如何通过标准输入输出流启动 MCP Server，实现与 LLM 客户端的通信
- **目标用户**：Claude Code、其他 MCP 客户端
- **预期产出**：稳定的 STDIO 传输通道

### 关键节点
1. **起点**：CLI 参数解析，识别 STDIO 模式
2. **中间步骤**：
   - 加载配置文件
   - 创建 StdioServerTransport
   - 连接 MCP Server
   - 等待客户端请求
3. **终点**：Server 监听 stdin，准备处理请求

### 角色/参与者
| 角色 | 职责 | 输入 | 输出 |
|------|------|------|------|
| CLI | 参数解析 | 命令行参数 | 启动模式 |
| ConfigLoader | 加载配置 | 配置文件路径 | Config 对象 |
| McpServer | 协议处理 | MCP 请求 | MCP 响应 |
| StdioTransport | 数据传输 | stdin 流 | stdout 流 |

### 流程/路径
- **正常路径**：配置加载 → Server 创建 → Transport 连接 → 等待请求
- **异常路径**：配置错误 → 日志记录 → 进程退出

### 数据流转
| 节点 | 输入数据 | 输出数据 | 传递方式 |
|------|----------|----------|----------|
| CLI | 命令行参数 | 启动模式 | 函数调用 |
| ConfigLoader | 配置路径 | Config | JSON 解析 |
| Server | Config | MCP Server 实例 | 构造函数 |
| Transport | Server | 双向流 | connect() |

### 异常/边界情况
| 异常类型 | 触发条件 | 处理方式 |
|----------|----------|----------|
| 配置文件不存在 | 文件路径错误 | 进程退出 + 错误日志 |
| 配置格式错误 | JSON 解析失败 | 进程退出 + 错误日志 |
| 端口冲突 | 已被占用 | 自动重试其他端口 |
