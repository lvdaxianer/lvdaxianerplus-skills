# HANDLER-004: Cache Handler 缓存管理

## 故事目标

- **核心问题**：验证 cache.handler.ts 能正确管理缓存数据和统计
- **目标用户**：Dashboard 用户
- **预期产出**：GET 获取缓存统计，DELETE 清空缓存

## 关键节点/里程碑

- **起点**：请求到达 /api/cache
- **中间步骤**：
  1. 根据方法执行操作
  2. GET → 获取缓存统计 (entries, hitRate, size)
  3. GET /entries → 获取所有缓存条目
  4. DELETE /:toolName → 删除特定工具缓存
  5. DELETE → 清空所有缓存
- **终点**：返回操作结果 JSON
