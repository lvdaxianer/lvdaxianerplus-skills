# 保存注意事项到 Context 文件

## Overview

当用户在项目中输入包含"注意事项："前缀的内容时，自动将注意事项保存到项目根目录下的 `.ai/` 目录中的 context 文件。

## When to Use

- 用户输入 "注意事项：xxx" 时
- 用户输入 "注意事项-模块名：xxx" 时

## 支持的格式

| 输入格式 | 保存位置 | 示例 |
|---------|---------|------|
| `注意事项：xxx` | `.ai/context.md` | 注意事项：这是通用注意事项 |
| `注意事项-模块名：xxx` | `.ai/模块名-context.md` | 注意事项-editor：编辑器相关注意 |

## Core Actions

### 1. 解析用户输入

检测输入是否匹配以下模式：

- **通用格式**: `注意事项：<内容>` → 保存到 `.ai/context.md`
- **模块格式**: `注意事项-<模块名>：<内容>` → 保存到 `.ai/<模块名>-context.md`

**正则表达式**:
```
^注意事项(?:-([^：]+))?：(.+)$
```

- 匹配组1: 模块名（可选）
- 匹配组2: 注意事项内容

### 2. 确定目标文件

1. 获取当前项目的根目录路径
2. 检查 `.ai` 目录是否存在，不存在则创建
3. 根据是否有模块名确定目标文件名：
   - 无模块: `context.md`
   - 有模块: `<模块名>-context.md`

### 3. 读取或创建文件

**如果文件不存在**，创建新文件并写入标题：

```markdown
# {模块名}注意事项

## {日期}

- {内容}
```

**如果文件已存在**，检查是否需要新的日期分组：

1. 读取文件内容
2. 查找最后一个 `##` 标题（日期）
3. 如果日期是今天 → 追加到该日期下
4. 如果日期不是今天 → 创建新的日期分组

### 4. 追加内容

使用以下格式追加注意事项：

```markdown
## 2026-03-13

- 这是一条注意事项
- 这是另一条注意事项
```

### 5. 写入文件

将更新后的内容写回文件，并告知用户保存结果。

## 文件格式示例

### 通用 context - .ai/context.md

```markdown
# 项目注意事项

## 2026-03-13

- 这是一个测试注意事项
- 这是另一个注意事项

## 2026-03-14

- 新增的注意事项
```

### 模块 context - .ai/editor-context.md

```markdown
# Editor 注意事项

## 2026-03-13

- 编辑器相关的注意事项
```

## 实现步骤

1. 使用正则表达式解析用户输入，提取模块名和内容
2. 获取当前工作目录作为项目根目录
3. 构建 `.ai` 目录的完整路径
4. 确保 `.ai` 目录存在
5. 根据模块名确定目标文件名
6. 读取目标文件（如存在）或创建新文件结构
7. 将注意事项追加到当天日期分组下
8. 写入文件并确认保存成功

## 错误处理

- 如果无法创建目录，提示用户检查权限
- 如果无法写入文件，提示用户检查文件权限
- 如果正则不匹配，忽略该输入（不处理）
- 如果用户输入 `注意事项：` 后面没有内容，提示用户输入有效内容

## 额外处理规则

### 1. 模块名验证

模块名需要做安全处理：
- 移除或替换不允许的文件名字符（如 `/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`）
- 限制模块名长度（最大 50 字符）
- 模块名首字母大写（如 `editor` → `Editor`）

### 2. 多行内容处理

如果用户输入包含换行：
- 将换行转换为空格连接成单行
- 或者在Markdown中使用 `>` 引用格式保留换行

### 3. 标题生成规则

- 无模块: `# 项目注意事项`
- 有模块: `# {模块名} 注意事项`（首字母大写）

### 4. 工作目录获取

使用 Claude Code 的工具（如 `Bash pwd` 或环境变量）获取当前工作目录作为项目根目录。
