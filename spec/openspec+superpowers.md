# OpenSpec + Superpowers 完整开发流程文档

## 一、文档说明

本文档适用于 Superpowers v5.x 版本（无斜杠命令，自然语言触发技能），详细梳理 OpenSpec 与 Superpowers 搭配的全流程，聚焦「接口契约标准化+前后端无缝对接」，适用于前后端分离项目、API 服务开发、快速迭代的场景。

核心定位：OpenSpec 定义「接口做什么、接口结构是什么」，Superpowers 执行「接口怎么实现、怎么同步」，全程无需手动输入斜杠命令，AI 自动触发对应技能，确保前后端、测试、AI 接口认知统一。

## 二、核心角色分工

- **OpenSpec**：契约制定者（甲方），负责定义「接口做什么、接口结构是什么」
  - 用 OpenAPI 规范定义接口路径、请求/响应结构、字段约束、状态码
  - 锁死接口边界，冻结后禁止自由修改
  - 任何变更必须先改契约，再同步前后端代码

- **Superpowers**：契约执行引擎（乙方），负责执行「接口怎么实现、怎么同步」
  - 读取 OpenSpec 契约，进入严格执行模式
  - 基于契约以 TDD 模式实现接口开发、生成前后端代码、完成校验
  - 全程不偏离契约，确保前后端无缝对接

## 四、注意事项

- 全程无需输入 /superpowers 斜杠命令，通过自然语言触发 Superpowers 技能即可。

- OpenSpec 契约冻结后，禁止随意修改接口结构；若需变更，必须先修改契约，再同步前后端代码。

- 每次接口变更后，必须执行全链路同步和审计，确保契约与代码、文档完全对齐。
